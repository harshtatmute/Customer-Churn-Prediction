Customer Churn Prediction - Project Flow

1. Problem Definition
   - Business Problem: Telecom companies lose revenue when customers cancel (churn).
     Goal: Predict whether a customer will churn (Yes/No) based on their profile & usage.
     ML Task: Binary Classification (Churn = Yes / No).


2. Data Collection
   - Dataset: Telco Customer Churn (Kaggle)

3. Exploratory Data Analysis (EDA)
   - Check dataset shape, info, missing values
   - Visualize churn distribution
   - Univariate & bivariate analysis

4. Data Preprocessing
   - Drop 'customerID'
   - Convert 'TotalCharges' to numeric
   - Handle categorical variables (One-Hot Encoding)
   - Scale numerical features

5. Model Building
   - Baseline: Logistic Regression, Decision Tree
   - Advanced: Random Forest, XGBoost

6. Model Evaluation
   - Accuracy, Precision, Recall, F1-score
   - Confusion Matrix
   - ROC-AUC Curve

7. Model Saving
   - Save best model using joblib

8. Deployment (Optional)
   - Build a Streamlit app
   - Input: Customer details
   - Output: Churn probability

9. Documentation
   - Write README.md with results, plots, and usage guide
